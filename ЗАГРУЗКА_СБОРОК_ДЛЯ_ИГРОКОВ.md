# Как загрузить сборку на сервер и играть вместе

## Для администратора сервера

### Шаг 1: Подготовка сборки

У вас должна быть готовая сборка сервера (с модами/плагинами) в формате `.jar` файла.

**Популярные типы сборок:**
- **Forge** - для модов
- **Fabric** - для модов
- **Paper/Spigot** - для плагинов
- **Другие** - любые кастомные сборки

### Шаг 2: Загрузка сборки на хостинг

Загрузите ваш `.jar` файл на один из сервисов:

#### Вариант A: GitHub Releases (РЕКОМЕНДУЕТСЯ)

1. Создайте репозиторий на GitHub (или используйте существующий)
2. Создайте **Release**:
   - Перейдите в **Releases** → **Create a new release**
   - Укажите версию (например: `v1.0`)
   - Загрузите файл `.jar`
   - Опубликуйте Release
3. Получите прямую ссылку:
   - Нажмите правой кнопкой на файл `.jar` в Release
   - Выберите **"Copy link address"**
   - Пример: `https://github.com/username/repo/releases/download/v1.0/server.jar`

#### Вариант B: Другие хостинги

- **Dropbox**: Загрузите файл, получите прямую ссылку
- **Google Drive**: Настройте прямую ссылку
- **Собственный хостинг**: Разместите файл и получите URL

### Шаг 3: Настройка на Railway

1. В Railway откройте ваш проект
2. Перейдите в **Settings → Variables**
3. Добавьте переменную:
   - **Имя**: `CUSTOM_SERVER_URL`
   - **Значение**: `https://ваш-url-к-сборке.jar`
   - Пример: `https://github.com/username/repo/releases/download/v1.0/server.jar`
4. Сохраните изменения
5. **Перезапустите сервер** в Railway

### Шаг 4: Проверка

После перезапуска проверьте логи:
- Должно появиться: "Скачивание кастомной сборки Minecraft сервера..."
- Затем: "Кастомная сборка успешно скачана!"
- Сервер должен запуститься

## Для игроков (как подключиться)

### Шаг 1: Узнайте версию и моды сервера

Попросите администратора сервера сообщить:
- **Версию Minecraft** (например: 1.20.1)
- **Тип сборки** (Forge, Fabric, Paper и т.д.)
- **Список модов/плагинов** (если нужны)
- **Ссылку на сборку** (если нужно скачать моды для клиента)

### Шаг 2: Установите нужную версию Minecraft

1. Откройте **Minecraft Launcher**
2. Перейдите в **Installations** (Установки)
3. Нажмите **New Installation** (Новая установка)
4. Выберите версию, которая совпадает с сервером
5. Сохраните

### Шаг 3: Установите моды (если нужны)

#### Для Forge сервера:

1. Скачайте **Forge** для нужной версии Minecraft:
   - https://files.minecraftforge.net/
   - Выберите версию, совпадающую с сервером
   - Скачайте **Installer** (установщик)

2. Установите Forge:
   - Запустите установщик
   - Выберите **Install client**
   - Установите в папку Minecraft

3. Скачайте моды:
   - Попросите администратора список модов
   - Скачайте те же версии модов, что на сервере
   - Поместите в папку `mods/` в профиле Minecraft

4. Запустите игру с профилем Forge

#### Для Fabric сервера:

1. Скачайте **Fabric Loader**:
   - https://fabricmc.net/
   - Выберите версию Minecraft
   - Скачайте установщик

2. Установите Fabric:
   - Запустите установщик
   - Выберите версию Minecraft
   - Установите

3. Скачайте моды:
   - Попросите администратора список модов
   - Скачайте те же версии модов
   - Поместите в папку `mods/` в профиле Fabric

4. Запустите игру с профилем Fabric

#### Для Paper/Spigot (плагины):

- Обычно **не нужны моды на клиенте**
- Просто используйте стандартную версию Minecraft
- Плагины работают только на сервере

### Шаг 4: Подключитесь к серверу

1. Откройте Minecraft с правильным профилем (Forge/Fabric/стандартный)
2. Нажмите **Multiplayer** (Сетевая игра)
3. Нажмите **Add Server** (Добавить сервер)
4. Введите адрес сервера (получите у администратора)
5. Подключитесь!

## Важные замечания

⚠️ **Версии должны совпадать!**
- Версия клиента должна совпадать с версией сервера
- Моды должны быть тех же версий, что на сервере

⚠️ **Моды обязательны для Forge/Fabric**
- Если сервер с модами, клиент тоже должен иметь эти моды
- Без модов вы не сможете подключиться

⚠️ **Плагины не нужны на клиенте**
- Если сервер на Paper/Spigot, моды на клиенте не нужны
- Плагины работают только на сервере

## Создание сборки для игроков

Если вы администратор и хотите упростить процесс для игроков:

### Вариант 1: Создайте модпак

1. Используйте **CurseForge** или **Modrinth**:
   - Создайте модпак с нужными модами
   - Игроки смогут установить его одним кликом

2. Или создайте архив:
   - Заархивируйте папку `.minecraft` с модами
   - Игроки распакуют и будут играть

### Вариант 2: Подготовьте инструкцию

Создайте документ с:
- Ссылкой на сборку сервера
- Списом модов с версиями
- Пошаговой инструкцией установки
- Ссылками на скачивание

## Пример полной настройки

### Администратор:

1. Создает сборку Forge 1.20.1 с модами: JEI, JourneyMap, Waystones
2. Загружает на GitHub Releases
3. Указывает `CUSTOM_SERVER_URL` на Railway
4. Перезапускает сервер

### Игрок:

1. Устанавливает Forge 1.20.1
2. Скачивает моды: JEI, JourneyMap, Waystones (те же версии)
3. Помещает в папку `mods/`
4. Запускает игру с профилем Forge
5. Подключается к серверу

## Решение проблем

### "Incompatible mods" (Несовместимые моды)
- Убедитесь, что версии модов совпадают с сервером
- Проверьте версию Minecraft

### "Connection refused" (Подключение отклонено)
- Проверьте адрес сервера
- Убедитесь, что сервер запущен

### "Missing mods" (Отсутствуют моды)
- Скачайте все моды, которые есть на сервере
- Проверьте версии модов

### Игра не запускается
- Проверьте версию Java (нужна Java 17 или 21)
- Убедитесь, что Forge/Fabric установлены правильно
