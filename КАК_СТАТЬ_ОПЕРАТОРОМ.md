# Как стать оператором на сервере Minecraft

## Где находится файл ops.json

Файл `ops.json` находится в контейнере по пути:
```
/minecraft/ops.json
```

Если Volume примонтирован к `/minecraft`, файл сохраняется там.

## Способы стать оператором

### Способ 1: Через команду в игре (если уже есть оператор)

Если на сервере уже есть оператор:
1. Попросите его выполнить в игре: `/op ваш_ник`
2. Готово!

### Способ 2: Создать файл ops.json через Railway CLI

**Важно:** Railway CLI не предоставляет прямой доступ к контейнеру через `railway shell`. Нужен другой подход.

#### Вариант A: Через Railway Web Interface (если доступен)

1. В Railway откройте ваш проект
2. Найдите раздел для работы с файлами (если доступен)
3. Отредактируйте файл напрямую

#### Вариант B: Через команду в игре (РЕКОМЕНДУЕТСЯ)

**Самый простой способ:**
1. Подключитесь к серверу в Minecraft
2. Если вы первый игрок, попробуйте выполнить: `/op ваш_ник`
3. Если не работает, используйте способ ниже

#### Вариант C: Создать файл через переменные окружения

Можно создать файл при запуске сервера, но это требует изменения скрипта запуска.

6. **Добавьте ваш UUID и имя:**
   ```json
   [
     {
       "uuid": "ваш-uuid",
       "name": "ваш_ник",
       "level": 4,
       "bypassesPlayerLimit": false
     }
   ]
   ```

**Как узнать UUID:**
- Используйте сайт: https://mcuuid.net/
- Введите ваш никнейм Minecraft
- Скопируйте UUID

### Способ 3: Через переменную окружения (при первом запуске)

Можно настроить автоматическое добавление оператора при первом запуске, но это требует изменения скрипта запуска.

### Способ 4: Через игру (если вы первый игрок)

Если вы первый игрок на сервере и сервер только что создан:
1. Подключитесь к серверу
2. В некоторых случаях первый игрок автоматически становится оператором
3. Если нет - используйте один из способов выше

## Формат файла ops.json

Пример содержимого:
```json
[
  {
    "uuid": "12345678-1234-1234-1234-123456789abc",
    "name": "ваш_ник",
    "level": 4,
    "bypassesPlayerLimit": false
  },
  {
    "uuid": "87654321-4321-4321-4321-cba987654321",
    "name": "другой_игрок",
    "level": 4,
    "bypassesPlayerLimit": false
  }
]
```

**Параметры:**
- `uuid` - уникальный идентификатор игрока (обязательно)
- `name` - никнейм игрока (обязательно)
- `level` - уровень прав (1-4, обычно 4 для полных прав)
- `bypassesPlayerLimit` - может ли обходить лимит игроков (true/false)

## Проверка, что вы оператор

После добавления в ops.json:
1. Перезапустите сервер в Railway (если редактировали файл)
2. Подключитесь к серверу
3. Попробуйте выполнить команду: `/op`
4. Если вы оператор, увидите список операторов
5. Попробуйте команду: `/gamemode creative` - должна работать

## Альтернатива: Использовать команду /op в игре

**Самый простой способ** - если на сервере уже есть оператор:
- Попросите его выполнить: `/op ваш_ник`
- Не нужно редактировать файлы!

## Важные замечания

⚠️ **UUID обязателен** - имя игрока может измениться, но UUID остается постоянным

⚠️ **Перезапуск** - после редактирования ops.json нужно перезапустить сервер

⚠️ **Безопасность** - операторы имеют полный доступ к серверу, добавляйте только доверенных игроков

⚠️ **Volume** - если Volume не примонтирован к `/minecraft`, файл ops.json будет теряться при перезапуске

## Быстрый способ (если есть доступ к Railway CLI)

```bash
# Подключитесь к контейнеру
railway shell

# Перейдите в папку minecraft
cd /minecraft

# Создайте файл ops.json (замените UUID и имя)
cat > ops.json << 'EOF'
[
  {
    "uuid": "ваш-uuid-здесь",
    "name": "ваш_ник",
    "level": 4,
    "bypassesPlayerLimit": false
  }
]
EOF

# Перезапустите сервер в Railway
```

## Получение UUID

1. Зайдите на https://mcuuid.net/
2. Введите ваш никнейм Minecraft
3. Скопируйте UUID (формат: `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`)
