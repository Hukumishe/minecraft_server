# Промпт для Lovable - Управление Minecraft сервером

Создай современное веб-приложение для управления Minecraft сервером на Railway с темной темой в стиле Minecraft.

## Основные страницы:

1. **Dashboard** (`/`) - главная страница со статусом сервера, быстрыми действиями и графиками
2. **Загрузка сборки** (`/upload`) - форма для загрузки кастомных .jar файлов через URL
3. **Настройки мира** (`/world-settings`) - настройки при создании мира (seed, режим игры, сложность, PvP и т.д.)
4. **Управление сервером** (`/server`) - запуск/остановка, просмотр логов, адрес для подключения
5. **Сохранения** (`/backups`) - автоматическое и ручное сохранение мира, история бэкапов

## Функциональность:

### Загрузка сборки:
- Input поле для URL кастомной сборки (.jar)
- Кнопка проверки доступности файла
- Сохранение в переменную окружения Railway `CUSTOM_SERVER_URL`

### Настройки мира:
- Seed (опционально)
- Режим игры: Survival/Creative/Adventure/Spectator
- Сложность: Peaceful/Easy/Normal/Hard
- Toggle: PvP, генерация структур, Nether, End
- Максимальное количество игроков (по умолчанию 5)
- Расстояние прорисовки и симуляции (sliders)
- Сохранение в переменные окружения Railway

### Управление сервером:
- Статус сервера (зеленый/красный/желтый индикатор)
- Кнопки: Запустить/Остановить/Перезапустить
- Логи в реальном времени с автообновлением
- Отображение TCP Proxy адреса с кнопкой копирования
- Информация: версия, игроки онлайн, uptime, память

### Сохранение мира:
- Toggle: автосохранение при выходе всех игроков
- Toggle: автосохранение при остановке сервера
- Интервал автосохранения (5/10/15/30 минут)
- Кнопка ручного сохранения
- Список сохранений с датой, размером, кнопками скачать/восстановить
- Настройка максимального количества сохранений

## Технический стек:
- React 18+ с TypeScript
- Tailwind CSS или shadcn/ui для UI
- Zustand для state management
- React Query для API запросов
- Axios для HTTP запросов
- React Hook Form + Zod для форм

## Интеграция с Railway API:
- Получение статуса: `GET /v1/services/{serviceId}`
- Обновление переменных: `PATCH /v1/variables/{variableId}`
- Получение логов: `GET /v1/deployments/{deploymentId}/logs`
- Перезапуск: `POST /v1/deployments/{deploymentId}/restart`
- Получение доменов: `GET /v1/services/{serviceId}/domains`

## Дизайн:
- Темная тема в стиле Minecraft (зеленые акценты)
- Современные карточки и компоненты
- Адаптивный дизайн
- Toast уведомления для действий
- Индикаторы загрузки

## Логика автосохранения:
1. Мониторинг количества игроков через логи/API
2. При количестве игроков = 0 → выполнить `/save-all` → создать бэкап
3. При остановке сервера → выполнить `/save-all` → создать финальный бэкап
4. По расписанию → выполнять `/save-all` с заданным интервалом

Начни с создания структуры приложения, навигации и основных страниц. Используй современный UI с темной темой.
