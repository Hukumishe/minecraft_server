# Как добавить несколько операторов на сервер

## Настройка нескольких операторов

Скрипт поддерживает до **5 операторов** через переменные окружения.

## Переменные окружения для операторов

В Railway: **Settings → Variables** добавьте следующие переменные:

### Первый оператор (обязательно, если нужны операторы):
- **Имя**: `OPS_UUID`
- **Значение**: UUID первого оператора

- **Имя**: `OPS_NAME`
- **Значение**: Никнейм первого оператора

### Второй оператор:
- **Имя**: `OPS_UUID_2`
- **Значение**: UUID второго оператора

- **Имя**: `OPS_NAME_2`
- **Значение**: Никнейм второго оператора

### Третий оператор:
- **Имя**: `OPS_UUID_3`
- **Значение**: UUID третьего оператора

- **Имя**: `OPS_NAME_3`
- **Значение**: Никнейм третьего оператора

### Четвертый оператор:
- **Имя**: `OPS_UUID_4`
- **Значение**: UUID четвертого оператора

- **Имя**: `OPS_NAME_4`
- **Значение**: Никнейм четвертого оператора

### Пятый оператор:
- **Имя**: `OPS_UUID_5`
- **Значение**: UUID пятого оператора

- **Имя**: `OPS_NAME_5`
- **Значение**: Никнейм пятого оператора

## Пример настройки для двух операторов

Добавьте в Railway Variables:

```
OPS_UUID = 12345678-1234-1234-1234-123456789abc
OPS_NAME = Player1

OPS_UUID_2 = 87654321-4321-4321-4321-cba987654321
OPS_NAME_2 = Player2
```

## Получение UUID

Для каждого игрока:

1. Зайдите на https://mcuuid.net/
2. Введите никнейм игрока
3. Скопируйте UUID (формат: `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`)

## После настройки

1. **Перезапустите сервер** в Railway (Settings → Restart Service)
2. При первом запуске создастся файл `ops.json` со всеми операторами
3. Все указанные игроки автоматически станут операторами

## Проверка

После перезапуска:

1. Подключитесь к серверу в Minecraft
2. Попробуйте выполнить команду: `/op`
3. Должен показаться список всех операторов
4. Попробуйте команду: `/gamemode creative` - должна работать

## Формат создаваемого файла

Скрипт автоматически создаст `ops.json` в формате:

```json
[
  {
    "uuid": "uuid-первого-оператора",
    "name": "ник-первого-оператора",
    "level": 4,
    "bypassesPlayerLimit": false
  },
  {
    "uuid": "uuid-второго-оператора",
    "name": "ник-второго-оператора",
    "level": 4,
    "bypassesPlayerLimit": false
  }
]
```

## Важные замечания

⚠️ **Порядок не важен** - можно добавить только второго оператора, пропустив первого (но лучше начинать с первого)

⚠️ **UUID обязателен** - имя игрока может измениться, но UUID остается постоянным

⚠️ **Перезапуск обязателен** - после добавления переменных нужно перезапустить сервер

⚠️ **Volume важен** - если Volume примонтирован к `/minecraft`, файл `ops.json` сохранится и не потеряется

## Добавление операторов позже

Если нужно добавить оператора после создания `ops.json`:

1. **Способ 1:** Используйте команду в игре (если уже есть оператор):
   - `/op ник_нового_оператора`

2. **Способ 2:** Добавьте переменные окружения и удалите `ops.json` из Volume (если есть доступ), затем перезапустите сервер

3. **Способ 3:** Используйте команду в игре для добавления, файл обновится автоматически

## Удаление оператора

Чтобы удалить оператора:

1. Используйте команду в игре: `/deop ник_оператора`
2. Или отредактируйте `ops.json` напрямую (если есть доступ)
